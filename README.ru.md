## Что такое ETCLOVG?

**ETCLOVG** — это семимерная таксономия для оценки и построения продакшн-систем ИИ-агентов.

| Слой | Название | Ключевая задача |
|:---:|:---|:---|
| **E** | Исполнение и песочница | Где выполняется код? 7 типов песочниц (Subprocess, Docker, WASM…) |
| **T** | Инструменты и протоколы | Как подключаются инструменты? MCP, A2A, Function Calling, OpenAPI |
| **C** | Контекст и память | Как агент запоминает? Трёхуровневая память с инкрементальным сжатием |
| **L** | Жизненный цикл | Как агент планирует и действует? PER-цикл, мультиагентная делегация |
| **O** | Наблюдаемость | Видим ли мы, что происходит? Трассировка, учёт затрат, heartbeat |
| **V** | Верификация | Корректен ли вывод? Предварительная проверка, пост-валидация, регрессия |
| **G** | Управление и безопасность | Безопасно ли это? 4-уровневая защита, очистка PII, конституционный ИИ |

## На основе анализа 170+ фреймворков

ETCLOVG Harness получен путём систематического анализа более **170 открытых фреймворков агентов**, включая GenericAgent, Hermes Agent, OpenClaw, Claude Code и CodeWhale.

## Использование

```python
from etclovg import ETCLOVGHarness

harness = ETCLOVGHarness(api_key="sk-...", model="gpt-4o")
result = harness.run("Расскажи о последних достижениях в области ИИ")
print(result["output"])
print(f"Стоимость: ${result['cost_estimate_usd']:.4f}")
```

## Архитектура

```
ETCLOVG Harness
├── E-слой: Песочница исполнения
├── T-слой: Мультипротокольный реестр инструментов
├── C-слой: Трёхуровневое управление контекстом
├── L-слой: Цикл План→Исполнение→Рефлексия
├── O-слой: Трассировка, затраты, heartbeat
├── V-слой: Валидация, верификация, регрессия
└── G-слой: Управление, очистка PII, аудит
```
